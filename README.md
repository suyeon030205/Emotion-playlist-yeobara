
<여바라-Emotion Playlist>

서비스 요약
Emotion Playlist – 표정을 찍으면, 지금 당신의 감정에 맞는 음악을 추천해주는 AI 서비스

주제 구분
-	S타입 현대인의 스트레스 해소를 위한 서비스

  
팀원 소개
팀명: 여바라
팀원: 
-	박수연: 프론트엔드 개발(로그인/회원과입 UI, 거울모드가 반영된 촬영화면, 감정 분석 결과의 시각화, 플레이리스트 재생화면)
-	김민서: 백엔드 개발(AI 서버 구축 및 관리, AI엔진 연동 및 데이터 전달, 유튜브 AIP호출 및 결과 처리)
-	진유민:Ai/데이터 매핑( 비디오 프레임 추출 및 감지, 프레임별 감정 분류 모델 통합, 순간 감정 평균화, 감정매핑 및 키워드 결정)
  
시연 영상
(필수) Youtube 링크 (선택) Github Repository 페이지에서 바로 볼 수 있도록 넣어주셔도 좋습니다.

서비스 소개
서비스 개요
-	Emotion Playlist는 사용자가 카메라를 1~2초간 응시하는 것만으로 표정의 미세한 변화를 AI가 자동 포착하고,
DeepFace 모델로 감정을 정밀 분석한 뒤, 분석된 감정 데이터(주감정/감정 비율)를 기반으로 현재 심리를 가장 잘 해소하거나 전환할 수 있는 음악을 추천해주는 ‘AI 감정 진단 & 뮤직 테라피 서비스’이다.
-	복잡한 클릭 없이, 그저 카메라를 바라보는 단 하나의 행동만으로
감정 진단 → 플레이리스트 추천이 완성되는 초간단 UI가 특징이다. 


타서비스와의 차별점
1. 능동적 입력이 필요없는  인터페이스
기존에 존재하던 감정 관리 앱들은 일기 작성, 자가 테스트, 체크리스트 등 사용자의 참여를 요구한다. 하지만 Emotion Playlist는
카메라를 보는 것 단 하나로 모든 과정이 자동으로 진행된다.
2. 감정 표현 불능 상태에서도 사용가능 
감정 명명 능력이 떨어지거나 자신이 기분이 뭔지 모르겠는 순간에도 Ai 가 표정을 기반으로 감정을 정량화 해준다. 즉, 직접 “슬픈 플레이리스” 등을 검색할 필요없이 현재 감정에 따른 플레이리스트를 추천받을 수 있다. 


구현 내용 및 결과물
1.	로그인 /회원가입 기능
	로그인 페이지와 회원가입 페이지를 각각 HTML 구조로 작성, 로그인 성공시 메인 감정 분석 페이지로 이동하도록 구성
	Flask 기반의 사용자 인증 API 구축, 회원가입시 사용자 정보를 파일에 저장하는 로직 구현, 로그인 시 저장된 사용자 정보와 대조하여 인증처리 
2.	감정 분석 페이지 (촬영) 
	1초 웹캠 녹화 기능, Blob 형태로 녹화 데이터 추출해 FromData에 담아 서버로 전송, UI 상에 분석중 오버레이 화면 표시 
	프론트에서 보낸 영상파일을 수신, 영상 파일을 임시로 저장한 뒤 AI 분석 모듈로 전달, 감정 분석 후 결과를 JSON으로 정리해 프론트로 반환
	1초 영상에서 프레임을 일정 간격으로 추출, Deepface 모델로 각 프레임의 감정 분류 후 감정 확률값을 평균내어 dominant emotion 도출
	감정 결과를 기반으로 youtube 검색 키워드를 생성함
3.	분석 결과 화면
	분석된 감정 확률 값을 받아 게이지 바 UI 로 보여줌, youtube API 에서 전달된 영상 3개를 카드 형태로 제공
	생성된 키워드로 Youtube Search API 호출해서 영상 리스트 생성,  키워드/감정확률겂/영상들을 프론트로 전달
	감정 분석 결과에 따라 감정-키워드 매핑, 분석결과/키워드/youtube 검색 결과를 하나로 연결하도록 통합.
4.	달력 화면
	분석 결과 페이지에서 기록 보기 버튼을 누르면 감정 기록 달력으로 이동하도록 설정,  기록된 감정이 있는 날짜의 이모지를 클릭하면 오버레이 형식으로 상세 내용 표시 
	감정 분석 완료 될 때마다 백엔드에서 해당 사용자의 분석결과(dominant emotion  + 추천 노래 영상) 를 날짜별로 저장하는 API 구축, 
	감정 분석 결과에서 산출된 지배 감정 과 유튜브 1순위 결과를  하루 단위 데이터로 구조화

향후 개선 혹은 발전 방안
1.	감정 변화 패턴 기반 맞춤 추천 강화
•	사용자의 감정 기록이 누적될수록, AI가 개인의 패턴을 학습하여
같은 감정이라도 사용자 취향에 맞는 추천 플레이리스트를 제공한다.
•	예를 들어, 슬픔의 감정일 때 발라드 보다  피아노 곡을 더 좋아하는 사용자나, 분노를 느낄 때 락 보다는 팝을 듣고 싶어하는 사용자 등을 구별해서 개인별 알고리즘 추천이 가능해진다
•	장기적으로는 “오늘 감정 → 취향 → 최적 추천”의 모델로 발전시킬 수 있다.
2.	사용자별 감정 기록·통계 대시보드 제공
•	사용자가 서비스를 이용할 때마다 감정 분석 결과를 자동 저장하여,
일·주·월 단위 감정 변화 그래프를 생성한다. 
•	이를 바탕으로 사용자는 본인이 스트레스 받기 쉬운 시간대, 요일, 환경 등을 파악할 수 있으며 스스로 정서 관리 루틴을 만들 수 있다.
